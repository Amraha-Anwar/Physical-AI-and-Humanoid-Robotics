# Building Packages with rclpy

The standard way to organize ROS 2 code is in **Packages**. We will create a Python package to implement a simple "Heartbeat" system for our robot.

## 2.1 Creating a Workspace

First, create a dedicated workspace directory. This is where all your source code lives.

```bash
mkdir -p ~/ros2_ws/src
cd ~/ros2_ws/src
```

## 2.2 Creating a Python Package

We use the `ros2 pkg create` command.

```bash
ros2 pkg create --build-type ament_python my_humanoid_pkg --dependencies rclpy std_msgs
```

*   `ament_python`: Specifies the build system for Python.
*   `dependencies`: Pre-configures `package.xml` with necessary libraries.

## 2.3 Writing a Publisher Node

This node will simulate the robot's heartbeat, publishing a status message every second.

Create `~/ros2_ws/src/my_humanoid_pkg/my_humanoid_pkg/heartbeat_node.py`:

```python
#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class HeartbeatNode(Node):
    def __init__(self):
        super().__init__('heartbeat_node')
        self.publisher_ = self.create_publisher(String, 'robot_status', 10)
        self.timer = self.create_timer(1.0, self.timer_callback)
        self.get_logger().info('Heartbeat Node has started.')

    def timer_callback(self):
        msg = String()
        msg.data = 'System OK: All motors online'
        self.publisher_.publish(msg)
        self.get_logger().info(f'Publishing: "{msg.data}"')

def main(args=None):
    rclpy.init(args=args)
    node = HeartbeatNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

## 2.4 Writing a Subscriber Node

This node listens for the heartbeat and logs it.

Create `~/ros2_ws/src/my_humanoid_pkg/my_humanoid_pkg/monitor_node.py`:

```python
#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class MonitorNode(Node):
    def __init__(self):
        super().__init__('monitor_node')
        self.subscription = self.create_subscription(
            String,
            'robot_status',
            self.listener_callback,
            10)
        self.subscription  # prevent unused variable warning

    def listener_callback(self, msg):
        self.get_logger().info(f'Monitor received: "{msg.data}"')

def main(args=None):
    rclpy.init(args=args)
    node = MonitorNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

## 2.5 Configuration and Building

1.  **Edit `setup.py`**: Register your nodes so ROS 2 can find them.
    ```python
    entry_points={
        'console_scripts': [
            'heartbeat = my_humanoid_pkg.heartbeat_node:main',
            'monitor = my_humanoid_pkg.monitor_node:main',
        ],
    },
    ```

2.  **Build the Workspace**:
    ```bash
    cd ~/ros2_ws
    colcon build --packages-select my_humanoid_pkg
    ```

3.  **Run**:
    ```bash
    source install/setup.bash
    ros2 run my_humanoid_pkg heartbeat
    ```

:::tip Debugging
Use `ros2 topic list` and `ros2 topic echo /robot_status` to inspect the data flowing through the system without writing a subscriber.
:::
