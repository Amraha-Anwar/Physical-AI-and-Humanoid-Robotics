# Gazebo Simulation Setup

RViz showed us the robot model, but it didn't apply gravity or collisions. For that, we need a **Physics Engine**. We use **Gazebo** (specifically the modern "Ignition" series, now just called Gazebo).

## 1.1 Installation

For ROS 2 Humble, the recommended version is **Gazebo Fortress**.

```bash
sudo apt-get update
sudo apt-get install ros-humble-ros-gz
```
This package installs Gazebo and the `ros_gz_bridge`, which translates ROS 2 messages to Gazebo's internal transport format.

## 1.2 The URDF vs. SDF Dilemma

Gazebo uses **SDF (Simulation Description Format)** natively. While modern Gazebo can ingest URDF, it's often better to explicitly convert or add SDF-specific tags to your URDF for better simulation stability.

### Adding Gazebo Tags to URDF

To make our `leg.urdf` works in simulation, we need to define properties like mass and inertia.

```xml
  <!-- Inertial Properties (Required for Physics) -->
  <inertial>
    <mass value="1.0"/>
    <inertia ixx="0.01" ixy="0.0" ixz="0.0" iyy="0.01" iyz="0.0" izz="0.01"/>
  </inertial>

  <!-- Gazebo Plugin -->
  <gazebo>
    <plugin filename="libgazebo_ros2_control.so" name="gazebo_ros2_control">
      <parameters>$(find my_humanoid_pkg)/config/controllers.yaml</parameters>
    </plugin>
  </gazebo>
```

## 1.3 Spawning the Robot

We use a launch file to start Gazebo and spawn our model.

Create `spawn_robot.launch.py`:

```python
import os
from ament_index_python.packages import get_package_share_directory
from launch import LaunchDescription
from launch.actions import IncludeLaunchDescription
from launch.launch_description_sources import PythonLaunchDescriptionSource
from launch_ros.actions import Node

def generate_launch_description():
    pkg_ros_gz_sim = get_package_share_directory('ros_gz_sim')
    
    # Start Gazebo
    gazebo = IncludeLaunchDescription(
        PythonLaunchDescriptionSource(
            os.path.join(pkg_ros_gz_sim, 'launch', 'gz_sim.launch.py')),
        launch_arguments={'gz_args': '-r empty.sdf'}.items(),
    )

    # Spawn Entity
    spawn = Node(
        package='ros_gz_sim',
        executable='create',
        arguments=['-topic', 'robot_description', '-name', 'my_bot', '-z', '0.5'],
        output='screen',
    )

    return LaunchDescription([gazebo, spawn])
```

## 1.4 Unity Visualization (Note)

While Gazebo handles the heavy physics, **Unity** can be used for high-fidelity visualization or VR interaction.
*   **Scope**: In this book, we strictly use the **ROS-TCP-Connector** to send TF (Transform) data from Gazebo -> ROS 2 -> Unity.
*   **Setup**: Import the URDF Importer package in Unity to mirror the Gazebo state visually. No physics calculations happen in Unity; it is purely a "viewer."
